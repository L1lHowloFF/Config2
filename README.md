# Config2
ДЗ по конфигу


## Этап 2. Сбор данных
Цель: реализована основная логика получения данных о зависимостях для их дальнейшего анализа и визуализации. Запрещено пользоваться менеджерами пакетов и сторонними библиотеками для получения информации о зависимостях пакетов. Требования:

Использован формат пакетов Rust (Cargo).
Извлечена информация о прямых зависимостях заданного пользователем пакета, используя URL-адрес репозитория.
(только для этого этапа) Выводятся на экран все прямые зависимости заданного пользователем пакета.
Результат выполнения этапа сохранен в репозиторий стандартно оформленным коммитом

# Описание функций и настроек
## Класс DependencyParser:
### Основные функции
__init__(self, config) - инициализация  
get_cargo_toml_from_github(self, repo_url) - скачивание Cargo.toml  
parse_dependencies(self, cargo_toml_content) - парсинг зависимостей  
get_direct_dependencies(self) - получение прямых зависимостей  

## Класс DependencyError:
### Основные функции
Исключение для ошибок зависимостей  

## main.py функции:
### Основные функции
print_dependencies(dependencies) - вывод списка зависимостей  
main() - обновлена для этапа 2
